## Goal
DX serializer LLM format cleanup (prefix elimination, legacy mode, `@=` compact syntax) + `--format` command with spaced/indented LLM output, then benchmark token efficiency against TOON, TONL, and Tauq

## Constraints & Preferences
- DX LLM format IS the human-friendly format â€” no more `@prefix` markers, comma-separated legacy mode, or `@=[]` compact syntax
- `--compact` stays but only means single-line sections, no more `@=` objects
- `--beautify` stays as human-readable `[section]` + aligned `=`
- `--format` is reformatted LLM: spaces around `=`, blank lines between entries, indented section rows â€” *not* human format
- Modes are mutually exclusive: beautify > format > compact
- Inline objects inside parens keep `field=value` (no spaces) â€” parser expects no whitespace after `=`
- Leaf inlining (`::`), key abbreviation dictionary, and `SerializerConfig` kept as-is

### Overall Ranking (cl100k tokens, lower = better)
| Rank | Format | Bytes | cl100k tokens | vs JSON |

### DX Serializer vs Competitors
- **vs JSON**: 48.8% fewer tokens
- **vs TOON**: 10.0% fewer tokens (15,026 vs 16,693)
- **vs Tauq**: 12.4% fewer tokens (15,026 vs 17,148)
- **vs TONL**: 11.2% fewer tokens (15,026 vs 16,915)
- **vs DX Serializer (--format)**: default is 7% more efficient (15,026 vs 17,079)

### Key Findings
- DX Serializer beats all competitors across every dataset size and shape
- On small mixed data (configs, projects, CI pipelines): DX Serializer is **23% better than TOON**, **69% better than TONL**
- On large tables (1000 rows): DX Serializer is **9% better than TOON**, **12% better than TONL**
- `--compact` is identical to default (< 0.03% difference)
- `--format` adds ~7% tokens for readability but still beats all other formats
