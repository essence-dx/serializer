# DX Benchmarks

Benchmarks measuring DX Compact's **token efficiency** and **retrieval accuracy** compared to JSON, JSONC, YAML, TOON, XML, and CSV.

> [!NOTE]
> All `.dx` files in this folder are generated by the `@dx-serializer/core` Bun encoder.
> The `showcase/extreme/` folder contains record-breaking benchmarks achieving up to −86% vs JSON.

## Quick Start

```bash
# Install dependencies
cd benchmarks
bun install

# Run token efficiency benchmark (requires @dx-serializer/core + gpt-tokenizer)
bun benchmark:tokens

# Run retrieval accuracy benchmark (requires API keys)
BENCHMARKS_DIR=. DRY_RUN=true bun run scripts/accuracy-benchmark.ts
```

## Token Efficiency Benchmark

Measures token count reduction across JSON, JSONC, YAML, TOON, and DX Compact:

1. Convert datasets to all formats
2. Tokenize using `gpt-tokenizer` (`o200k_base` encoding)
3. Calculate savings and generate report

## Project Structure

```
├── *.json                    # JSON source data (pretty-printed)
├── *.jsonc                   # JSON compact (minified)
├── *.dx                      # DX Compact (encoder-generated, round-trip verified)
├── *.toon                    # TOON (official @toon-format/cli)
├── *.yaml                    # YAML equivalents
├── data/
│   └── github-repos.json     # Top 100 GitHub repos

│   ├── README.md             # Showcase benchmark results

│       └── ...               # More extreme-scale datasets
├── results/
│   ├── token-efficiency.md   # Token savings report
│   └── retrieval-accuracy.md # Accuracy report
├── scripts/
│   ├── token-efficiency-benchmark.ts
│   └── accuracy-benchmark.ts
└── src/                      # Benchmark source code
```

## Key Results

| Comparison | Average | Best Case |
|-----------|---------|-----------|
| DX vs JSON pretty | **−59%** | **−86%** |
| DX vs JSON compact | **−37%** | **−75%** |
| DX vs YAML | **−39%** | **−60%** |
| DX vs TOON | **−34%** | **−51%** |

DX Compact beats **every format on every benchmark file** (20/20 files vs TOON).
